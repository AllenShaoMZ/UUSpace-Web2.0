## UDP 转发封装

软件端 `DataRelayService.TryUnwrapForwardFrame()` 会先判断 UDP 包是否带 `URLY` 封装头，再把内部载荷交给遥测解析：

| 版本 | 头部结构 | Sheet 判断 |
| --- | --- | --- |
| V1 | `URLY` + 版本 + 原始端口 + 原始 Sheet + 内部长度 + 载荷 | 优先用封装头 Sheet |
| V2 | V1 + UTC 毫秒时间戳 | 优先用封装头 Sheet，并用封装时间作为曲线时间 |
| V3 | `URLY` + 版本 + UTC 毫秒时间戳 + 载荷 | 头里没有端口/Sheet，按当前监听端口 7101-7108 映射 |

Web 端现在与软件端一致：先解 `URLY`，再解析内部 payload；连接管理仍显示 `rawLength/rawHex` 便于排查外层封装。

## 指令区分逻辑

桌面端指令表核心列来自 `CommadTableSource`：指令代号、指令名称、数据源码、目标端口号、目标 IP、DataSrc。IDE 指令树的分类规则在 `MainForm.cs`：

| 规则 | Web 分类 |
| --- | --- |
| 指令代号以 `K` 开头 | 星上指令 |
| 指令代号以 `DLX` 或 `D` 开头 | 动力学指令 |
| 其他代号 | 其他指令 |

指令 `Type` 数字按软件端 `μCOSControlPackage.ApidType` 选项映射，例如 `0=间接指令`、`1=内部指令`、`3=立即软件注数`、`7=连续地址注数单包`。`DataSrc=-1` 表示可参数化模板，`DataSrc=1` 表示固定源码。
