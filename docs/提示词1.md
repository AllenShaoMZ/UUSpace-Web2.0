【问题1：视图切换卡顿】
当前 renderView() 每次都执行 $("#stage").innerHTML = renderer()，完整销毁并重建整个 DOM 树。对于遥测表格（几百行）和曲线视图，这导致明显卡顿。

优化方案：
- 在 bindViewActions() 中移除 window.__UUSPACE_BOUND 这个有缺陷的保护机制，改为每次 renderView() 后无条件（重新）绑定事件委托到 #stage
- 对于遥测表格视图，不要整体重渲染，改为只更新变化的单元格（使用 document.getElementById 精准更新）
- 切换视图时先检查 state.activeView 是否真的改变了，没改变不执行完整 renderView

【问题2：搜索框需要点多次才能输入】
当前 input 事件触发 renderView() 后，搜索框 DOM 节点被销毁重建，焦点丢失，用户体验极差。

优化方案：
- 搜索过滤不要触发 renderView()，改为直接在现有 DOM 上对表格行做 show/hide（参考已有的 filterRows() 函数）
- 输入完成（300ms 防抖）后才执行一次完整 renderView()
- 渲染完成后立即恢复焦点：const el = document.getElementById('paramSearch'); if (el) { el.focus(); el.setSelectionRange(el.value.length, el.value.length); }

请按上述方案修改 bindViewActions() 和 renderView() 相关逻辑，保持其他功能不变。
