# M5 — 指令界面

| 属性 | 内容 |
|------|------|
| **阶段** | P3（高）+ P4（中）+ P5（低） |
| **依赖** | M0（`/api/command/send`）、MC（可选） |
| **主要文件** | `app.js`（`renderCommandCenter`、`sendCommand`）、`realtime.js`（`mapCommandRow`） |

## 目标

在**现有指令卡片 + 详情 + 两步确认** Web UI 上，增加指令链、参数化发送、执行日志与链编辑；组包逻辑对齐桌面业务，界面不参照桌面。

## Vibe Coding 任务清单

### P3 — 高优先级

- [ ] **CMD-03** 前端日志环形队列（最多 **500** 条），含时间、代号、目标、结果、HEX 摘要
- [ ] **CMD-03** 指令页底部或折叠区「执行日志」列表，成功/失败样式区分
- [ ] **CMD-01** 启动时 `GET /api/cmdchain`，解析 `commandIds` 与 `weight` → `delayMs`（**毫秒**，Q-06）
- [ ] **CMD-01** 顶部「指令链」下拉 +「执行链」按钮（Q-11）
- [ ] **CMD-01** `ChainRunner`：顺序调用现有 `sendCommand` 逻辑，步间 `await sleep(delayMs)`
- [ ] **CMD-01** 任一步失败 → **中止整条链** + 日志标红 + 提示（Q-07）
- [ ] **CMD-02** `DataSrc === '-1'` 时弹出参数表单（字段来自 `rawRow` / 指令表同行，E 列判定）
- [ ] **CMD-02** `buildCommandPacket(rawRow, formValues)` → HEX，再走两步确认
- [ ] **CMD-02** 禁止对 `-1` 指令直接发送占位 `data: "0"`

### P4 — 中优先级

- [ ] **CMD-04** 链执行时显示进度条 `当前步/总步`
- [ ] **CMD-05** 链编辑 UI（增删改顺序、delayMs），未保存时执行前提示（Q-11）
- [ ] **CMD-05** `PUT /api/cmdchain` 写回 `Commad/cmdchain.txt`（服务端实现）

### P5 — 低优先级（可选）

- [ ] **CMD-06** 多选指令 + 间隔批量发送
- [ ] **CMD-07** 指令收藏 + 筛选入口，持久化 `command.favorites`

### 约束

- [ ] 保持 `segment` 分类、`command-card` 网格、两步确认发送
- [ ] 不新增桌面式 MDI 指令窗口

### 验收

- [ ] 执行 `cmdchain.txt` 样例链（如「飞轮测试」），步骤间延时可见，日志逐步增加
- [ ] 选一条 `DataSrc=-1` 指令，填参后 HEX 变化且发送成功
- [ ] 链中故意失败一条时，后续步骤不再执行
