# 卫星地面综测系统（Web版）UI 重构方案 v2.0

> 定位：专业级实时遥测 + 遥控指令 Mission Workspace
> 场景：局域网多工作站访问 / 长时间值守 / 高信息密度
> 风格：Mission Control Workspace（航天地面控制中心风格）
> 核心目标：从“后台管理系统”升级为“任务控制台”

---

# 一、为什么重构

现有方案沿用了 WinForms + 后台管理系统布局：

```text
左导航 + 中间主内容 + 右侧面板 + 底栏
```

存在问题：

### 1. 空间利用率低

左导航长期占据约220px。

对于遥测、曲线、大表场景属于浪费。

---

### 2. 页面切换成本高

用户需要：

实时曲线 ↔ 遥测大表 ↔ 指令发送

频繁切换。

认知割裂。

---

### 3. 像 ERP，而不是航天地面系统

大量：

* 表格
* 边框
* 卡片
* 菜单

导致：

视觉噪音严重。

---

### 4. 数据层级不明确

真正重要：

* 当前任务状态
* 告警
* 链路状态
* 实时趋势

却与大量UI元素抢视觉权重。

---

设计目标：

### 从：

页面驱动

切换页面完成任务

### 改为：

任务驱动

一个工作台完成任务

---

# 二、新设计理念

采用：

Mission Workspace（任务工作区）

参考：

* SpaceX Mission Control
* Bloomberg Terminal
* Unreal Dock
* VSCode 面板系统
* QGroundControl

核心思想：

```text
传统后台：

菜单 → 页面 → 功能


Mission Workspace：

任务 → 工作区 → 数据
```

# 七、曲线视觉重做

旧：

Grafana风

新：

Mission风

Echarts建议：

```javascript
grid:{
top:40,
left:30,
right:30,
bottom:30
}

lineStyle:{
width:2
}

areaStyle:{
opacity:.08
}
```

增加渐变：

```javascript
areaStyle:{
color:new echarts.graphic.LinearGradient(
0,
0,
0,
1,
[
{
offset:0,
color:'rgba(91,124,250,.35)'
},
{
offset:1,
color:'rgba(91,124,250,0)'
}
]
)
}
```

视觉效果：

```text
      ╱╲
_____/  ╲____
```

而不是：

```text
_________
```
