app.js 的 refreshUdpViews() 在每次收到 UDP 包时都调用 renderView()，当 UDP 速率 25pkt/s 时，遥测表格每秒被整体重渲染25次，导致严重卡顿。

请按以下方案重写 refreshUdpViews() 和表格数据更新逻辑：

1. 【分离"数据更新"和"DOM更新"】
   - syncPacketValues() 只更新 state.sheetLiveValues（已实现），不触发任何 DOM 操作
   - DOM 更新改为固定 4fps（每250ms）最多执行一次

2. 【表格行局部更新】
   实现 updateTableRowsInPlace() 函数：
   - 遍历当前页面上所有 tr[data-param-row]
   - 对每行，根据 state.sheetLiveValues 查找最新值
   - 只更新该行中"当前值"和"十六进制"对应的 td
   - 值有变化时给 td 加 .fresh-row 类（0.8s 后自动移除，CSS transition 实现）
   - 不重建任何 DOM 节点

3. 【状态栏单独更新】
   实现 updateSheetStatusLine() 只更新 .table-status-line 的内容

4. 【曲线视图不受影响】
   当 state.activeView === 'curve' 时，refreshUdpViews() 只推送曲线数据点，不触发任何 DOM 操作
