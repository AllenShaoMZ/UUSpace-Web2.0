以下是一个卫星地面综测 Web 系统（app.js + styles.css），请优化界面使操作更直观、信息层次更清晰。用户是卫星测控工程师，主要任务是监视遥测数值、发现异常、发送指令。

【优化1：状态总览页 renderStatus()】
- 顶部 mission-card 信息过多，轨道动画（orbit-visual）对工程师无实用价值，建议替换为实时连接状态矩阵（显示每个 UDP 端口的连接状态色块）
- stat-grid 中"协议规则数"对实时监视没有帮助，替换为"当前帧率"和"最近告警数"两个指标，用颜色区分正常/异常
- status 事件流每条加时间戳，不同级别（success/warning/danger）用左边框颜色区分而非右侧 tag

【优化2：遥测表格页 renderTelemetryTable()】
- Sheet 标签页当前8个一行，在小屏溢出，改为显示端口号和实时包计数，有数据的标签页加绿色边框高亮
- 表格左侧"已选波道"栏目与表格内容重复，建议折叠为悬浮抽屉（点击工具栏按钮展开）
- 表格行当前值列：数值型参数用小型迷你 sparkline 展示趋势（最近10个值），非数值型参数正常显示文字
- 告警状态参数行高亮整行背景（淡红色），而不只是状态文字变色

【优化3：指令控制页 renderCommandCenter()】
- 指令卡片当前信息太密，只显示：指令代号、名称、类别标签、发送按钮。其余详情点击后在右侧面板展示
- 发送按钮改为分两步：第一次点击"确认发送"高亮，第二次点击才真正发送。防止误触
- 发送结果在卡片底部用动画提示（成功绿色✓，失败红色×，1.5秒后消失）

请只修改对应的 render*() 函数和 styles.css 中相关样式，不改变数据逻辑。
