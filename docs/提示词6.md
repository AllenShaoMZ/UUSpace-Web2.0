这是一个卫星地面综测系统的前端代码（app.js 约1500行，styles.css 约900行）。当前主要架构问题：

核心问题：
1. 没有虚拟DOM或diff机制，每次状态变化都通过 innerHTML 完整重建视图
2. 事件绑定依赖全局 window.__UUSPACE_BOUND 标志，在视图切换后有残留问题
3. UDP 数据推送（25Hz）直接触发 renderView()，导致高频DOM重建
4. 搜索框输入触发全量渲染导致焦点丢失，用户体验差

请在不引入任何构建工具、不使用 npm/webpack 的前提下（纯原生 JS），对 app.js 进行以下重构：

【第一步：引入轻量级状态订阅机制】
实现一个简单的 createStore(initialState) 函数：
- state 变更通过 store.set(key, value) 进行
- 各视图通过 store.subscribe(keys, callback) 订阅感兴趣的 state key
- 只有被订阅的 key 变化时，对应视图的 callback 才被调用

【第二步：DOM更新分层】
- 层1（全量渲染）：视图切换时触发，renderView() 保持现有逻辑
- 层2（区域更新）：UDP数据刷新时只更新表格行数值、状态栏、Dock摘要
- 层3（动画帧）：曲线 canvas 在 requestAnimationFrame 中独立运行，与DOM操作完全解耦

【第三步：搜索独立处理】
所有搜索框实现"输入→即时过滤（隐藏/显示DOM节点）→300ms防抖→状态同步"，不触发 renderView()

保持所有现有功能、API 接口（/api/*）、CSS 类名不变，保证升级后与现有 styles.css 和 realtime.js 完全兼容。
请分步骤展示修改，先修改 bindViewActions()，再修改 refreshUdpViews()，最后修改搜索相关逻辑。
